# RideFlow iOS Demo

Personal iOS demo project built in SwiftUI as a student application exercise for a mobility-related product domain.

The goal of this project is to demonstrate my approach to:
- SwiftUI app structure
- feature-based project organization
- state management
- reusable UI components
- basic persistence
- testing and CI
- product thinking for a connected mobility use case

## Overview

This app is a concept/demo application focused on a smart ride experience, including ride tracking, routes, battery-related views, weather context, widgets, and app flow handling.

It is not intended as a production-ready product, but as a practical project that reflects my current iOS skills and my motivation to continue improving as an iOS engineer.

## Features

- Onboarding flow
- Main app navigation
- Ride screen with basic metrics
- Routes feature
- Battery-related views
- Weather mock integration
- Local route persistence
- Widget support
- Basic tests
- GitHub Actions CI setup

## Tech Stack

- Swift
- SwiftUI
- Xcode
- UserDefaults
- XCTest
- GitHub Actions

## Project Structure

- `App` – app entry, app state, navigation flow
- `Core` – shared models and utilities
- `Features` – feature modules
- `Services` – persistence and service layer
- `UI` – reusable UI components
- `Widgets` – widget-related functionality
- `Tests` – unit/integration tests

## Why I built this

I created this project on my own initiative as a practical iOS exercise aligned with the type of products and challenges I am interested in.

My intention was to build a domain-relevant demo that combines mobile UI, application structure, and feature thinking in a mobility-oriented scenario.
